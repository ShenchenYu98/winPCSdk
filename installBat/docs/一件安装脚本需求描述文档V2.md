# opencode插件一件安装脚本需求说明

---

## 一、需求说明

### 前提条件
默认用户已安装好opencode

### 需求内容
帮我生成一个一件安装的bat脚本文件，在windows系统内双击即可运行，满足以下功能：
1、将此脚本所在同文件夹内的`@wecode`文件夹整体内容复制到`C:\Users\USERPROFILE\.cache\opencode\node_modules`文件夹内
  若node_modules文件夹内已存在`@wecode`文件夹，则直接覆盖替换;

2、读取`C:\Users\USERPROFILE\.config\opencode`内的`opencode.json`文件，修改其中plugin字段的值
  默认`opencode.json`文件存在；
  若`plugin`字段不存在则新建该字段，其值是一个数组，新加值`[`@wecode/skill-opencode-plugin`]`;
  若`plugin`字段存在，但值中无`@wecode/skill-opencode-plugin`，则新加;
  若`plugin`字段存在，且值中有`@wecode/skill-opencode-plugin`，则无需覆盖，无需处理;
  
  `opencode.json`文件内容参考
  ```json
  {
    "$schema": "https://opencode.ai/config.json",
    "provider":{},
    "plugin": ["file:///D://featProject//message-bridge-opencode-plugin","@wecode/skill-opencode-plugin"],
  }
  ```

3、读取`C:\Users\USERPROFILE`内的`.npmrc`文件，进行字段修改
  对应字段若已存在，则进行覆盖，若不存在则进行新增；
  ```bash
  sslVerify=false
  strict-ssl=false
  @wecode:registry=https://cmc.xxx.com/npm/product_npm/
  ```

4、读取`C:\Users\USERPROFILE\.config\opencode`内的`message-bridge.json`文件，根据用户输入的ak,sk的值替换其中对应字段的值;
  若`message-bridge.json`文件不存在则新建；
  若`message-bridge.json`文件已存在，则复用该文件；
  若`message-bridge.json`中已存在ak，sk字段，则将用户输入的ak，sk值覆盖替换；
  bat脚本输入ak，sk值支持用户手动输入，也支持快捷键粘贴；
  输入ak，sk时要有良好的交互体验；

  `message-bridge.json`文件内容参考
  ```json
  {
    "xxx": "test",
    "auth": {
      "ak": "Ivk1/NQO5",
      "sk": "ME4CA454ASFAF"
    }
  }
  ```

5、输入完ak，sk后，提示用户即将修改的几项配置信息，经过用户同意输入`y`后执行，若输入`n`则不执行，提示用户安装失败；
message-bridge.json配置修改
1、ak:`用户输入值`;
2、sk:`用户输入值`;

.npmrc配置项修改:
1、sslVerify=false
2、strict-ssl=false
3、@wecode:registry=https://cmc.xxx.com/npm/product_npm/
