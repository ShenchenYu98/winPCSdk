# opencode-CUI — Skill 对话集成方案

## Context

现有聊天工具（类 DingTalk/Feishu/Slack）的 IM 消息仅支持纯文本，无法承载 AI 编码工具的富文本输出（代码块、Markdown、流式响应等）。

**方案**：不新建独立 AI 对话窗口，改为通过 IM 已有的 "/" 命令框架触发 **Skill 交互**——用户选择 SKILL 后，IM 聊天框上方出现微型状态条（IM客户端提供），可点击展开为 Skill 小程序，在其中与 AI 工具进行多轮对话。对话结果可选中发送回 IM 聊天。

IM客户端和小程序调用 **Skill SDK 插件**，IM客户端和小程序通过 `Skill SDK 插件` 与Skill 服务端交互建立连接。两者之间通过 WebSocket 长连接传递流式数据。

本次实现**仅考虑 IM客户端，Skill 小程序和Skill Server之间的交互，采用本地验证**，个人私有会话。Claude Code / OpenClaw / 远程模式均不在本次范围内。

| 组件 | 性质 | 运行位置 | 职责 |
|---|---|---|---|
| **Skill SDK 插件** | SDK插件 | IM 客户端 | 负责IM客户端，Skill 小程序和Skill Server之间的交互；作为IM客户端和Skill 小程序之间的交互； |

- IM 聊天客户端（已有，我们往里集成 Skill Sdk 插件）

- "/" 命令框架（已有，IM客户端提供）
- Skill 服务端（已有，Skill 服务端提供）
- Skill 小程序（已有）
- IM 聊天窗微型状态条Mini Bar（已有，IM客户端提供）

| 维度 | 内容 |
|---|---|
| 目标用户 | 使用现有聊天工具的开发者 |

## 验收标准

- [ ] **AC-1**：用户在 IM 输入框输入 "/" 后直接展示 OpenCode SKILL，点击后 IM 聊天框显示一行高亮 SKILL 名称，用户可输入问题并点击"生成"
- [ ] **AC-2**：点击"生成"后 IM 聊天区出现 1 行高的 Skill Mini Bar，显示"OpenCode 处理中…"
- [ ] **AC-3**：点击 Mini Bar 的"展开"按钮后，Skill 小程序覆盖 IM 聊天窗，可继续多轮对话
- [ ] **AC-4**：AI 响应以流式方式逐字显示，代码块有语法高亮和"复制"按钮
- [ ] **AC-5**：用户可选中 OpenCode 返回的任意文本，点击"发送到聊天"将其作为 IM 消息发送
- [ ] **AC-6**：重新打开 Skill 小程序后，历史会话和消息从数据库恢复
- [ ] **AC-7**：现有 IM 聊天功能不受任何影响（无回归）

### 用户交互流程

```
1. 用户在 IM 输入框输入 "/"
   → 直接展示 OpenCode SKILL（已有 "/" 框架）

2. 用户点击 OpenCode SKILL
   → IM 聊天框出现一行高亮的 SKILL 名称文本（如 "✦ OpenCode"）
   → 用户在 IM 输入框中继续输入问题

3. 用户点击 IM 聊天框的"生成"按钮
   → IM 聊天区出现 1 行高的 Mini Bar：[🔄 OpenCode 处理中…  ▢ 展开]
   → 点击Mini Bar中的展开后，Skill 小程序启动

4. OpenCode 流式响应
   → 原始事件透传：Skill Server → Skill 小程序
   → Skill 小程序解析 OpenCode 事件，渲染流式内容
   → Mini Bar 更新状态

5. 用户点击 "▢ 展开"
   → Skill 小程序展开，覆盖 IM 聊天窗上方
   → 展示完整对话视图（Skill 小程序解析并渲染：流式文本、代码高亮、Tool Use 等）

6. 用户在 Skill 小程序中继续多轮对话

7. 用户选中 OpenCode 某段回复，点击 "发送到聊天"
   → Skill 小程序 → Skill Server → IM 平台 API
   → IM 聊天窗显示该消息

8. 用户点击 "收起"
   → Skill 小程序收起，IM 聊天恢复
   → Mini Bar 仍显示，可再次展开
```
