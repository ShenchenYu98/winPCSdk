## 二、产品定位

数字分身客户端 SDK 是 IM 客户端与分身服务端之间的同步中间层，用于承接数字分身会话状态、分身详情和在线状态相关能力，并将服务端确认结果同步给 IM 客户端。

本期目标是让 IM 客户端可以稳定获取并刷新数字分身相关状态，同时支持将 IM 侧的会话状态修改同步至服务端，供其他端感知和保持一致。

---

## 三、产品边界

### 3.1 适用范围

- 仅面向 Windows IM 客户端场景
- 仅支持数字分身单聊场景
- 同一账号下，一个 `cloneId` 对应一个唯一分身会话
- SDK 对外以 `cloneId` 作为主标识
- 不同账号之间状态、监听和缓存需相互隔离

### 3.2 非本期范围

- 删除会话
- 标为已读
- 分身资料编辑
- 群聊场景
- 一分身多会话
- 全量会话列表接口
- 差量字段同步
- 失败回滚责任定义
- 实现方案、技术选型、协议细节

---

## 四、本期需求范围

本期仅包含以下 4 项能力：

1. 服务端分身会话状态同步到 IM 客户端
2. IM 客户端主动设置分身会话状态并同步到服务端
3. 按 `cloneId` 获取分身详情
4. 按 `cloneId` 监听单个分身在线状态

---

## 五、业务规则

### 5.1 主数据源规则

- 服务端是分身会话状态和分身详情的最终权威来源
- SDK 是同步中间层，不作为最终裁决方
- IM 当前显示状态若与服务端最新状态不一致，以服务端结果覆盖

### 5.2 会话模型规则

- 一个 `cloneId` 在同一账号下仅对应一个唯一分身会话
- SDK 对会话的查询、同步和监听均围绕 `cloneId` 进行

### 5.3 账号隔离规则

- 不同账号的分身会话状态不可互相复用
- 不同账号的在线状态监听结果不可互相透出
- 账号切换后，新账号仅可读取和接收本账号上下文下的数据

## 六、功能需求

### FR-1 服务端分身会话状态同步到 IM

#### 需求说明

当服务端侧数字分身会话状态发生变化时，SDK 需要将最新状态同步给 IM 客户端，供 IM 刷新对应分身会话。

#### 功能要求

- IM 启动时，SDK 需先从服务端获取当前分身完整会话状态，再提供给 IM 做初始化
- 运行过程中，只要服务端状态发生变化，SDK 需主动通知 IM
- SDK 同步给 IM 的数据口径为完整 `cloneSessionInfo`
- 名称、头像也纳入本期会话刷新范围
- 即使服务端推送的值与 IM 当前显示一致，SDK 仍需通知 IM
- 本期不定义断线后的状态补齐规则

#### 同步结果要求

- IM 可基于 SDK 返回的完整会话状态直接刷新当前分身会话展示
- 若 IM 当前显示与服务端最新状态不同，IM 需以 SDK 同步过来的服务端结果覆盖当前显示

### FR-2 IM 主动设置分身会话状态并同步服务端

#### 需求说明

当 IM 客户端主动修改分身会话状态时，SDK 需要将对应状态同步到服务端，以便服务端继续同步到其他端。

#### 功能要求

- IM 修改后，SDK 负责将状态转发到服务端
- 本期会话状态字段范围按现有 `cloneSessionInfo` 全量口径定义
- 本期至少包含会话展示和刷新所需的完整状态字段，不仅限于置顶和免打扰
- SDK 不向 IM 承诺成功或失败结果
- SDK 不承担失败回滚责任

#### 结果边界

- SDK 的职责是完成状态转发
- 服务端是否确认、其他端何时感知，不作为 SDK 在本期内向 IM 的承诺项

### FR-3 获取分身详情

#### 需求说明

SDK 需支持 IM 按 `cloneId` 获取数字分身详情，用于分身资料展示。

#### 功能要求

- 可按 `cloneId` 查询单个分身详情
- 本期详情字段仅包含：
  - 名称
  - 头像
  - 简介
- 本期不返回 `AK/SK`、英文名或其他扩展字段

### FR-4 监听单个分身在线状态

#### 需求说明

SDK 需支持 IM 按 `cloneId` 监听指定分身的在线状态变化。

#### 功能要求

- 监听粒度为单个分身
- 在线状态不并入会话状态同步链路，单独定义
- 本期状态值仅定义为：
  - 在线
  - 离线
- 注册监听成功后，SDK 需先返回该分身当前状态
- 后续当在线状态发生变化时，SDK 继续通知 IM

## 七、数据范围定义

### 7.1 会话状态数据范围

本期“完整会话状态”按现有 `cloneSessionInfo` 全量口径定义。  
SDK 同步给 IM 时，返回的是该分身当前完整会话快照，而不是仅返回变化字段。

### 7.2 分身详情数据范围

本期“分身详情”字段范围固定为：

- 名称
- 头像
- 简介

### 7.3 在线状态数据范围

本期“在线状态”字段范围固定为：

- 在线
- 离线

---

## 八、关键场景

### 8.1 IM 启动初始化

1. IM 启动后，需要初始化某个分身会话展示
2. SDK 从服务端获取该分身当前完整会话状态
3. SDK 将完整会话状态返回给 IM
4. IM 完成该分身会话初始化展示

### 8.2 服务端状态变化回刷 IM

1. 服务端中某个分身会话状态发生变化
2. SDK 感知到该变化
3. SDK 将该分身最新完整会话状态主动同步给 IM
4. IM 刷新对应分身会话展示

### 8.3 IM 修改会话状态

1. IM 修改某个分身会话状态
2. SDK 接收该修改并转发给服务端
3. 服务端后续可将结果同步至其他端

### 8.4 查看分身详情

1. IM 按 `cloneId` 请求分身详情
2. SDK 返回该分身的名称、头像和简介
3. IM 完成资料展示

### 8.5 监听分身在线状态

1. IM 按 `cloneId` 注册在线状态监听
2. SDK 先返回该分身当前在线状态
3. 当该分身在线状态后续发生变化时，SDK 持续通知 IM

---

## 九、验收标准

- IM 启动后可通过 SDK 获取分身当前完整会话状态并完成初始化
- 服务端任一分身会话状态变化后，IM 可收到 SDK 主动通知并完成刷新
- 名称、头像变化会触发会话刷新
- IM 修改分身会话状态后，SDK 可将状态转发到服务端
- IM 可按 `cloneId` 获取名称、头像、简介
- IM 可按 `cloneId` 监听单个分身在线/离线状态，且注册后先收到当前值
- 账号切换后，状态和监听结果不会串号

---

## 十、默认前提

- 服务端已具备会话状态查询能力
- 服务端已具备会话状态变更推送能力
- 服务端已具备分身详情查询能力
- 服务端已具备在线状态推送能力
- IM 已具备分身会话 UI 刷新能力
- IM 已知需要初始化的 `cloneId`
- 服务端是最终权威源，SDK 是同步中间层
