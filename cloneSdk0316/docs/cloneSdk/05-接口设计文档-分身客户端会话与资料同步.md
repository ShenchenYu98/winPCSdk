# 分身客户端会话与资料同步接口设计文档

**版本**: v1.0  
**日期**: 2026-03-16  
**状态**: 待确认  
**项目代号**: Clone Client Sync

我现在要做一个数字分身客户端SDK，它用于IM客户端和数字分身之间的交互，IM客户端和分身服务端之间的交互。
1、从服务端获取到的分身会话状态信息(分身名称，分身头像，会话置顶，会话免打扰等)要同步给IM客户端进行对应分身会话状态的刷新；
2、IM客户端主动设置分身会话状态(会话置顶，会话免打扰等)要同步给到分身服务端，从而同步到其他端(如PC端同步到移动端)；
3、提供接口支持获取分身详情(名称，头像，简介)；
4、提供接口能监听获取分身在线状态；

## 二、接口设计原则

1. 服务端最终确认结果为准，本地接口不承诺 IM 发起值一定成为最终状态。
2. IM 启动或重连后，更新全量会话状态。分身客户端只提供数据获取接口，不负责时机触发。
3. 分身客户端对 IM 输出“领域事件 + 最新确认快照”。
4. 同步失败即时返回，不设计后台自动重试接口。
5. 本期不定义具体 IPC 技术、HTTP 路径和存储引擎，只锁定接口语义和数据结构。

## 三、接口说明

### 3.1 根据数字分身ID获取会话信息

#### 接口说明

根据数字分身ID`cloneId`获取会话信息

#### 接口名

```typescript
getCloneSessionInfo(params:cloneSessionParams): Promise<cloneSessionInfo>
```

#### 调用方

分身客户端

#### 入参

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `cloneId` | string | 是 | 数字分身ID |

#### 出参

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `cloneId` | string | 是 | 数字分身ID |
| `avatar` | string | 是 | 数字分身头像地址 |
| `name` | string | 是 | 数字分身名称 |
| `nameEn` | string | 是 | 数字分身英文名称 |
| `htmlContent` | string | 是 | 数字分身会话最后一条消息描述(中文) |
| `htmlContentEn` | string | 是 | 数字分身会话最后一条消息描述(英文) |
| `createTime` | string | 是 | 数字分身会话消息展示时间 |
| `topType` | string | 是 | 数字分身会话是否置顶，"0"未置顶，"1"置顶 |
| `unReadCount` | number | 是 | 数字分身会话未读消息数 |
| `disturbType` | string | 是 | 数字分身会话是免打扰，"0"未免打扰，"1"免打扰 |
| `isMute` | int | 是 | 1显示红点，0显示具体的未读消息数，依赖unReadCount，若unReadCount为0依然不显示基数或红点 |
| `readState` | string | 是 | "1"表示数字分身会话已读，"0"表示未读 |
| `supportMarkTop` | number | 是 | "1"表示显示会话置顶入口，"0"表示不显示 |
| `supportDelete` | number | 是 | "1"表示显示会话删除入口，"0"表示不显示 |
| `needMuteIcon` | number | 是 | "1"表示显示会话静音喇叭图标，"0"表示不显示 |

#### 返回示例

```json
{
  "cloneId": "clone_9001",
  "avatar": "https://xxx",
  "name": "分身小白",
  "nameEn": "fenshen xiaobai",
  "htmlContent": "你好，我是小白，有什么能帮忙你的?",
  "htmlContentEn": "Hello,I am xiaobai? Can I help you?",
  "createTime": "1773653795",
  "topType": "1",
  "unReadCount": 12,
  "disturbType": "0",
  "isMute": "0",
  "readState": "0",
  "supportMarkTop": "1",
  "supportDelete": "1",
  "needMuteIcon": "1"
}
```

#### 实现说明
1、IM客户端调用window.Pedestal.callMethod('method://cloneSdk/getCloneInfo',payload)，此方法由IM客户端实现;
  该方法调用到分身客户端实际调用的是分身客户端中的`getCloneSessionInfo`接口
2、调用服务端 REST API 获取分身会话状态信息：
   - **URL**: `GET /api/skill/getCloneSessionInfo`
   - **查询参数**:
     ```json
     {
      "cloneId": "clone_9001",
     }
     ```


### 3.2 更新分身会话状态信息

#### 接口名

```typescript
updateCloneSessionInfo(params:cloneSessionStatus): Promise<void>

interface cloneSessionStatus {
  cloneId: "clone_9001",
  topType: "1",
  disturbType: "0",
  isMute: "0",
  readState: "0",
}
```

#### 接口说明

IM客户端更新会话状态后(如设置会话置顶,会话免打扰等)，同步该数据到分身客户端，然后同步到分身服务端

#### 入参

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `cloneId` | string | 是 | 数字分身ID |
| `topType` | string | 是 | 数字分身会话是否置顶，"0"未置顶，"1"置顶 |
| `disturbType` | string | 是 | 数字分身会话是免打扰，"0"未免打扰，"1"免打扰 |
| `isMute` | int | 是 | 1显示红点，0显示具体的未读消息数，依赖unReadCount，若unReadCount为0依然不显示基数或红点 |
| `readState` | string | 是 | "1"表示数字分身会话已读，"0"表示未读 |

#### 出参

无

#### 实现说明
1、IM客户端更新会话状态信息后调用window.Pedestal.callMethod('method://cloneSdk/updateCloneSessionInfo',payload);该方法为IM实现，是调用分身客户端接口的入口
2、window.Pedestal.callMethod('method://cloneSdk/updateCloneSessionInfo',payload)实际调用的是分身客户端中的`updateCloneSessionInfo`接口
3、调用服务端 REST API 更新会话状态信息：
   - **URL**: `POST /api/skill/updateCloneSessionInfo`
   - **请求体**:
     ```json
     {
      "cloneId": "clone_9001",
      "topType": "1",
      "disturbType": "0",
      "isMute": "0",
      "readState": "0",
     }
     ```

### 3.3 获取分身详情信息

#### 接口名

```typescript
getCloneInfo(params:queryCloneInfoParams): Promise<cloneInfo>

interface queryCloneInfoParams {
  cloneId: "clone_9001",
}

interface cloneInfo {
  cloneId: "clone_9001",
  avatar: "https://xxx",
  name: "分身小白",
  nameEn: "fenshen xiaobai",
  description: "分身简介",
  ak: "qasfafafa",
  sk: "popopopoasfsfa"
}
```

#### 接口说明

调用服务端接口，获取分身详情，返回给IM客户端进行对应信息渲染。

#### 入参

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `cloneId` | string | 是 | 数字分身ID |

#### 出参

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `cloneId` | string | 是 | 数字分身ID |
| `avatar` | string | 是 | 数字分身的头像资源 |
| `name` | string | 是 | 数字分身中文名称 |
| `nameEn` | string | 是 | 数字分身英文名称 |
| `description` | string | 是 | 数字分身简介 |
| `ak` | string | 是 | 数字分身的认证AK |
| `sk` | string | 是 | 数字分身的认证SK |

#### 实现说明
1、IM客户端获取分身详情时会调用window.Pedestal.callMethod('method://cloneSdk/getCloneInfo',payload);该方法为IM实现，是调用分身客户端接口的入口
2、window.Pedestal.callMethod('method://cloneSdk/getCloneInfo',payload)实际调用的是分身客户端中的`getCloneInfo`接口
3、调用服务端 REST API 更新会话状态信息：
   - **URL**: `GET /api/skill/getCloneInfo`
   - **查询参数**:
     ```json
     {
      "cloneId": "clone_9001",
     }
     ```

### 3.4 监听数字分身在线状态

#### 接口名

```typescript
onCloneStatus(params: onCloneStatusParams): Promise<void>

interface onCloneStatusParams {
  cloneId: string;
  callback: (result: CloneStatus) => void;
}

interface CloneStatus {
  status: "online" | "offline"
}
```

#### 接口说明

通过监听回调告知IM客户端分身是否在线，并触发对应回调

#### 入参

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `cloneId` | string | 是 | 数字分身ID |
| `callback` | function | 是 | 状态变更回调函数 |

#### 出参

无

#### 实现说明
1、监听WebSocket的message事件，根据事件类型(分身状态变化事件)触发对应传参回调


## 非对外接口能力

### 更新分身会话状态信息广播
1、监听WebSocket的message事件，根据事件类型(分身会话状态变更事件)触发事件广播
2、事件广播：window.Pedestal.sendBroadcast('broadcast://cloneSdk/updateCloneSession',payload);
  IM客户端提供的跨进程广播能力，分身客户端可以直接调用，payload就是要更新的分身会话信息，即服务端返回的分身会话状态信息；
3、触发事件广播后，在分身客户端进行 分身会话状态详情 本地数据缓存，缓存时根据`cloneId`进行隔离
